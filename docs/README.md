## Documentation Conventions

### Mermaid Diagrams

Architecture diagrams use Mermaid syntax and render in GitHub/GitLab. For local viewing, use a Mermaid-compatible markdown previewer or VS Code extension.

### Code Examples

All code examples are in TypeScript/TSX and follow the project's style guidelines.

### Icelandic/English

- UI strings and user-facing content: Icelandic
- Code, comments, and documentation: English
- Variable names: English (camelCase)

## Updating Documentation

When making changes to the codebase:

1. Update relevant documentation files
2. Ensure code examples match actual implementation
3. Update API documentation for new/changed interfaces
4. Keep architecture diagrams in sync with structure changes
